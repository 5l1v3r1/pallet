Documentation

Pallet documentation is maintained as org-mode files, which are exposed through the GitHub wiki.

Documentation updates should be made through the wiki or directly in the wiki repository.

Updating the wiki locally

You can clone the wiki to work on it locally. 

git clone git@github.com:pallet/pallet.wiki.git

If you want to view committed changes locally before pushing, then run you can run a local gollum server. This requires that you have a working ruby and rubygems installation. 

gem install gollum
gollum

Update the documentation site

To generate the documentation site, we pull the changes from the wiki repo, run autodoc, marginalia and publish the org project.

Merge wiki changes

The first time you do this, you will need to add a remote for the wiki in your clone of the pallet repository. 

git remote add wiki git@github.com:pallet/pallet.wiki.git

You can then pull in changes from the wiki. 

git pull wiki master

Publishing the org-mode project

The `org-mode` project is defined in doc/project.el. Open this file in emacs and run M-x eval-buffer. You will also need an open clojure repl, which can be set up correctly by connecting to a slime session started with mvn clojure:swank in the pallet-all project. With this repl running, you can run M-x org-publish to publish the html files.

Run autodoc

Autodoc is generated using: 

mvn clojure:autodoc

Run Marginalia

Marginalia documentation is generated using: 

mvn clojure:marginalia

Commit the generated documentation

In order to get the generated documentation onto the GitHub Pages site, the documentation needs to be committed and pushed. 

cd autodoc
git add -A
git commit -m "Updated documentation"
git push origin gn-pages
